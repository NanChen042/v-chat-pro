# 多账号测试指南

为了完整测试实时聊天功能（如好友申请、即时消息推送等），你需要模拟多个用户同时在线。以下是几种推荐的测试方法：

## 1. 浏览器多窗口/无痕模式 (最推荐)
由于本地存储（Token 和用户信息）是按域名隔离的，常规的多开窗口会共享同一个登录状态。
- **窗口 A**: 使用常规浏览器窗口登录账号 1。
- **窗口 B**: 打开**无痕模式 (Incognito/Private)** 窗口，登录账号 2。
- **结果**: 两个窗口拥有独立的 Session，可以互相发送消息并观察实时推送。

## 2. 多个浏览器同时使用
- 使用 **Chrome** 登录账号 1。
- 使用 **Safari** 或 **Edge** 登录账号 2。
- 这种方式同样能确保两人互不干扰。

## 3. 使用 `chat-test.html` (快速调试)
在项目根目录下有一个 `chat-test.html` 文件：
1.  直接在浏览器中打开此文件。
2.  输入不同的 `User ID`（例如 `1` 和 `2`）。
3.  点击“发送”即可模拟对应 ID 的用户发出消息。
4.  **注意**: 此工具绕过了 UI 和认证逻辑，仅用于快速验证后端 WebSocket 推送是否正常。

## 4. 多账号注册与准备
1.  启动前端和后端。
2.  在前端登录页点击“立即注册”，注册两个账号（如 `test1`, `test2`）。
3.  分别按上述方法登录，即可开始测试好友申请流程。

---

## 退出登录建议
如果你在测试中发现登录状态错乱，请点击“我”页面的“退出登录”按钮。它会清空本地的所有缓存数据，确保下次登录是一个干净的环境。
