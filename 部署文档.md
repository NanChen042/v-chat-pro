# 实时聊天系统部署文档（GitHub + Render + Vercel + MySQL）

本文档用于将当前项目完整上线并可正常使用前后端。

项目结构：
- 后端：`chat-server`（Nest.js + TypeORM + Socket.io）
- 前端：`v-chat`（Vue 3 + Vite + Vant + Pinia）

推荐部署组合：
1. 代码托管：GitHub
2. 数据库：Railway MySQL（或任意云 MySQL）
3. 后端服务：Render
4. 前端服务：Vercel

---

## 一、部署前准备

## 1.1 推送代码到 GitHub
在项目根目录执行：

```bash
git add .
git commit -m "feat: ready for production deployment"
git push origin main
```

## 1.2 后端改为可配置 CORS（必须）

编辑文件：`chat-server/src/main.ts`

将：
```ts
app.enableCors();
```

替换为：
```ts
app.enableCors({
  origin: process.env.CORS_ORIGIN?.split(',') || '*',
  methods: 'GET,HEAD,PUT,PATCH,POST,DELETE',
  credentials: true,
});
```

编辑文件：`chat-server/src/chat/chat.gateway.ts`

将：
```ts
@WebSocketGateway({
  cors: {
    origin: '*',
  },
})
```

替换为：
```ts
@WebSocketGateway({
  cors: {
    origin: process.env.CORS_ORIGIN?.split(',') || '*',
  },
})
```

## 1.3 前端增加 Vercel 路由回退（必须）

因为你使用 Vue Router history 模式，刷新子路由会 404，需添加 `vercel.json`。

在 `v-chat` 目录新建 `vercel.json`：

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

## 二、创建云数据库（Railway MySQL 示例）

1. 登录 Railway，创建 MySQL 服务。
2. 获取连接信息：
- `DB_HOST`
- `DB_PORT`
- `DB_USERNAME`
- `DB_PASSWORD`
- `DB_DATABASE`
3. 保存这些变量，后端部署时会用到。

> 注意：不要再使用本地 MySQL（localhost）作为线上数据库。

---

## 三、部署后端（Render）

## 3.1 创建 Render Web Service

1. 登录 Render
2. 点击 `New +` -> `Web Service`
3. 连接 GitHub 仓库
4. 选择当前仓库

## 3.2 Render 构建配置

- Root Directory: `chat-server`
- Runtime: `Node`
- Build Command:

```bash
corepack enable && pnpm install && pnpm run build
```

- Start Command:

```bash
pnpm run start:prod
```

## 3.3 Render 环境变量

在 Render 中添加以下环境变量：

```bash
DB_HOST=你的数据库Host
DB_PORT=3306
DB_USERNAME=你的数据库用户名
DB_PASSWORD=你的数据库密码
DB_DATABASE=你的数据库名
JWT_SECRET=请设置一个高强度密钥
NODE_ENV=production
CORS_ORIGIN=https://你的前端域名.vercel.app
```

> 如果你有多个前端域名（例如生产 + 预览），可用逗号分隔：
> `CORS_ORIGIN=https://a.vercel.app,https://b.vercel.app`

## 3.4 获取后端线上地址

部署成功后会得到类似地址：

```bash
https://your-backend.onrender.com
```

后续前端 `VITE_API_BASE_URL` 就填这个地址。

---

## 四、部署前端（Vercel）

## 4.1 创建 Vercel Project

1. 登录 Vercel
2. `Add New...` -> `Project`
3. 导入你的 GitHub 仓库
4. 配置项目

## 4.2 Vercel 构建配置

- Root Directory: `v-chat`
- Build Command: `pnpm build`
- Output Directory: `dist`

## 4.3 Vercel 环境变量

添加：

```bash
VITE_API_BASE_URL=https://your-backend.onrender.com
```

保存并部署。

## 4.4 获取前端线上地址

部署成功后会得到类似地址：

```bash
https://your-frontend.vercel.app
```

---

## 五、回填 CORS 并重启后端

前端地址确定后，返回 Render，把后端环境变量 `CORS_ORIGIN` 更新为真实前端域名：

```bash
CORS_ORIGIN=https://your-frontend.vercel.app
```

保存并触发一次 redeploy。

---

## 六、上线联调验收（必须逐项通过）

按以下步骤验证：

1. 打开前端线上地址
2. 注册用户 A
3. 注册用户 B
4. A 登录后搜索 B 并发送好友申请
5. B 登录后在“新的朋友”中通过申请
6. 验证 A 收到通过通知
7. A/B 双方进入聊天页互发消息
8. 验证首页未读数变化
9. 验证进入聊天页后已读清零

全部通过表示上线成功。

## 七、常见问题排查

## 7.1 前端请求还在打 localhost

现象：线上页面无法请求后端。

原因：`VITE_API_BASE_URL` 未配置或仍是本地地址。

处理：在 Vercel 环境变量中设置正确后端地址并重新部署。

## 7.2 后端 500 / 数据库连接失败

现象：注册登录报错。

原因：Render 上数据库环境变量错误。

处理：核对 `DB_HOST/DB_PORT/DB_USERNAME/DB_PASSWORD/DB_DATABASE`。

## 7.3 刷新页面 404（前端路由）

现象：直接打开 `/chat/2` 报 404。

原因：未配置 Vercel rewrite。

处理：确认 `v-chat/vercel.json` 存在并重新部署。

## 7.4 HTTP 正常但实时消息失败

现象：接口正常，聊天消息不实时。

原因：
1. 后端 Gateway CORS 未放行前端域名。
2. 前端 `VITE_API_BASE_URL` 与后端实际域名不一致。

处理：
1. 检查 `CORS_ORIGIN`。
2. 检查前端环境变量和后端 URL。
